# События CJM на лендинге NutriScan

Документ расшифровывает значение поля **`type`** и содержимое **`data`** для аналитики, отправляемой с одностраничного лендинга на `POST /app/stat-event`.

Реализация: `src/App.tsx` (эффект после монтирования приложения).

---

## Общие условия

| Условие | Описание |
|---------|----------|
| **Эндпоинт** | `{VITE_API_BASE_URL}/app/stat-event` |
| **Авторизация** | Заголовок `Authorization: Bearer <token>`. Токен берётся из ответа `POST /auth/login` и хранится в `localStorage` (`landing_auth_token`). Если токена нет, события **не отправляются** (кроме попыток до завершения логина). |
| **`durationSeconds`** | Секунды с момента **первой загрузки страницы** в этой вкладке (таймер стартует вместе с эффектом аналитики). |
| **`data`** | Всегда **строка JSON**. Внутри дополнительно к полям события обычно есть: `path`, `utm`, `authId` (`landing_auth_id` или `null`). |

---

## Справочник `type` (лендинг)

Все перечисленные значения относятся к префиксу **`cjm_landing_`** — это не маршруты SPA-приложения сканирования, а сценарий **одной HTML-страницы** с якорными секциями.

### `cjm_landing_open`

| | |
|---|---|
| **Что означает** | Пользователь открыл лендинг; зафиксировано начало сессии просмотра в браузере. |
| **Когда уходит** | Сразу после монтирования страницы и **повторно** примерно через **1,2 с** (чтобы событие успело уйти после асинхронного логина и появления токена). |
| **Зачем в аналитике** | Подсчёт визитов, сопоставление с `utm`, тип устройства, проверка доставки событий после авторизации. |

**Типичное содержимое `data` (после `JSON.parse`):**

| Поле | Тип | Описание |
|------|-----|----------|
| `href` | `string` | Полный URL страницы (`window.location.href`). |
| `referrer` | `string \| null` | `document.referrer`, если есть. |
| `isPhoneClient` | `boolean` | Эвристика «мобильный клиент» (User-Agent и/или узкий viewport), как для редиректа на приложение. |
| `path` | `string` | `pathname` (у лендинга обычно `/`). |
| `utm` | `string` | Значение query-параметра `utm` из URL (только метка, без префикса `utm=`). |
| `authId` | `string \| null` | Id пользователя с бэка (`user.id` из логина), если уже сохранён. |

---

### `cjm_landing_section_view`

| | |
|---|---|
| **Что означает** | Пользователь **впервые** довёл до достаточной видимости один из отслеживаемых блоков страницы (секцию или главный блок героя). |
| **Когда уходит** | Когда элемент пересекает порог **45%** видимой области (`IntersectionObserver`, `threshold: 0.45`). Повторно для того же `sectionId` **не** шлётся. |
| **Зачем в аналитике** | Воронка «дочитал ли до блока FAQ / до финального CTA», глубина скролла без привязки к точным пикселям. |

**Типичное содержимое `data`:**

| Поле | Тип | Описание |
|------|-----|----------|
| `sectionId` | `string` | Атрибут `id` у `<section>` или `<main>`. |
| `path` | `string` | Как выше. |
| `utm` | `string` | Как выше. |
| `authId` | `string \| null` | Как выше. |

---

### `cjm_landing_click`

| | |
|---|---|
| **Что означает** | Клик по интерактивному элементу **кнопка** или **ссылка** (`button`, `a`). |
| **Когда уходит** | На каждый такой клик на странице (делегирование на `document`). |
| **Зачем в аналитике** | Какие CTA и навигация используются, в каком блоке страницы. |

**Типичное содержимое `data`:**

| Поле | Тип | Описание |
|------|-----|----------|
| `elementTag` | `string` | `button` или `a`. |
| `elementText` | `string \| null` | Обрезанный текст элемента (`textContent.trim()`), может быть пустым у иконок. |
| `sectionId` | `string \| null` | `id` ближайшего предка `section[id]` или `header[id]` или `main[id]`. Если клик вне таких контейнеров — `null`. |
| `path` | `string` | Как выше. |
| `utm` | `string` | Как выше. |
| `authId` | `string \| null` | Как выше. |

---

### `cjm_landing_leave`

| | |
|---|---|
| **Что означает** | Пользователь покидает страницу или сворачивает вкладку. |
| **Когда уходит** | При `document.visibilityState === 'hidden'` или перед `beforeunload`. |
| **Зачем в аналитике** | Длительность сессии на лендинге, отличие «ушёл в другое приложение» от «закрыл вкладку». |

**Типичное содержимое `data`:**

| Поле | Тип | Описание |
|------|-----|----------|
| `reason` | `string` | `visibility_hidden` — вкладка скрыта / уход в фон; `beforeunload` — выгрузка страницы. |
| `path` | `string` | Как выше. |
| `utm` | `string` | Как выше. |
| `authId` | `string \| null` | Как выше. |

---

## Соответствие `sectionId` → блок на лендинге

Эти `id` участвуют в **`cjm_landing_section_view`** и могут попасть в **`cjm_landing_click`** как контекст клика.

| `sectionId` | Компонент / место на странице |
|-------------|------------------------------|
| `how` | Первый экран (hero): заголовок, подзаголовок, теги, кнопки «Пройти чекап» / «Как это работает». Тег: `<main class="hero" id="how">`. |
| `tech` | Блок «От состояния здоровья к рациону» (шаги процесса). |
| `food-impact` | Блок про влияние питания на организм. |
| `how-it-works` | Блок с видео «Посмотрите, как это работает». |
| `signals` | Блок «ИИ видит то, что не видит глаз». |
| `nutritionist` | Блок про обучение алгоритмов нутрициологом. |
| `faq` | Часто задаваемые вопросы. |
| `final-cta` | Финальный призыв «Персональный рацион…» и кнопка «Пройти чекап». |

Фиксированный **хедер** (`<header>`) в разметке без `id` в текущей версии — клики по пунктам меню в шапке могут дать `sectionId: null`, если клик не внутри `main#how`.

## Краткая шпаргалка по `type`

| `type` | Одной фразой |
|--------|----------------|
| `cjm_landing_open` | Зашёл на лендинг (старт + повтор через ~1,2 с). |
| `cjm_landing_section_view` | Первый раз увидел блок страницы по `sectionId`. |
| `cjm_landing_click` | Нажал кнопку или ссылку; есть текст и контекст секции. |
| `cjm_landing_leave` | Ушёл со страницы или спрятал вкладку (`reason`). |
